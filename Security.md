# Security Policy

## Supported Versions

Security updates are applied to the latest active code on the default branch.

## Reporting a Vulnerability

If you discover a security vulnerability, please:

1. **Do not open a public issue with exploit details**
2. Contact the maintainer privately (or open a minimal issue requesting secure contact)
3. Include:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Suggested mitigation (if known)

We will acknowledge reports as quickly as possible and work on a fix.
